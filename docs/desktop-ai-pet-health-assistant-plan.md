# 桌面 AI 桌宠 + 个人健康管理助手：落地方案

## 1. 你要做的产品可以拆成 6 个子系统

1. **桌面桌宠 UI（常驻）**
   - 显示角色动画、状态气泡、提醒。
   - 支持点击交互、语音唤醒、快捷操作（开始学习/开始运动/休息提醒）。

2. **多传感器采集层**
   - 摄像头：工作/学习状态检测、运动动作识别。
   - 无接触雷达：心率、呼吸率、在位状态。
   - 麦克风：语音输入。

3. **AI 感知与分析层**
   - 专注状态识别（是否离开座位、是否分心）。
   - 动作标准性评估（关节角度、节奏、稳定性）。
   - 健康趋势分析（运动量、心率区间、恢复情况）。

4. **教练与提醒层**
   - 实时语音反馈（“背挺直一点”“深蹲膝盖不要内扣”）。
   - 学习/工作节律提醒（番茄钟、久坐提醒、喝水提醒）。

5. **数据与隐私层**
   - 本地优先存储（SQLite + 本地加密）。
   - 仅上传可选的匿名统计。
   - 权限分级（摄像头/麦克风/雷达可独立开关）。

6. **语音对话层**
   - ASR（语音转文字）→ LLM（对话/建议）→ TTS（语音播报）。
   - 支持“教练模式”“陪伴模式”“安静模式”。

---

## 2. 推荐技术栈（先做 MVP，尽量稳）

### 桌面端（跨平台）
- **Tauri + React**（推荐）
  - 优点：体积小、性能好、易于调用 Rust/系统能力。
- 可选：Electron（生态成熟但更重）。

### 视觉与动作识别
- **MediaPipe Pose / MoveNet**：实时关键点检测，适合深蹲、俯卧撑、平板支撑等。
- **OpenCV**：视频流处理、背景和光照预处理。

### 雷达接入
- 依据硬件厂商 SDK（如串口/UART、USB CDC、BLE）。
- 采样后统一走 `Sensor Gateway`，转换成标准字段：
  - `heart_rate`, `resp_rate`, `presence`, `confidence`, `timestamp`。

### 语音链路
- 离线优先可选：
  - ASR：Whisper.cpp / Vosk
  - TTS：Piper
- 在线增强可选：云 ASR + 云 TTS + 大模型 API。

### 数据与分析
- **SQLite**（本地） + **DuckDB**（后期分析可选）。
- 时序数据表：心率、呼吸、动作评分、专注度。

## 3. 系统架构建议（可直接按模块开发）

- `desktop-app`：桌宠 UI、设置页、历史数据页。
- `sensor-service`：统一采集摄像头/雷达/麦克风，输出标准事件。
- `ai-service`：
  - 状态识别（学习/工作/离开）
  - 动作评估（姿态评分）
  - 建议生成（规则 + LLM）
- `coach-service`：提醒调度、语音播报、行为奖励机制。
- `data-service`：本地存储、加密、导出。

通信方式：本机 `WebSocket` 或 `gRPC`，事件总线模型（topic：`posture`, `vitals`, `focus`, `coach_alert`）。

---

## 4. 关键算法怎么做（从简单到高级）

### 4.1 工作/学习状态检测
先做规则版：
- 是否在画面中 + 是否持续坐姿 + 屏幕前停留时长。
- 检测到低头/趴桌/长时间不动时提醒。

再升级到模型版：
- 输入：姿态关键点序列 + 时间窗口（30~60s）。
- 输出：`focused`, `distracted`, `away`, `fatigued`。

### 4.2 运动动作标准性
- 用关键点算关节角度（髋、膝、肩、肘）。
- 每个动作定义“标准区间”：
  - 例如深蹲：
    - 下蹲时膝关节角度目标范围
    - 背部倾角上限
    - 膝盖内扣判定
- 输出：实时评分 + 纠正建议 + 计数。

### 4.3 心率呼吸融合
- 雷达给出心率/呼吸基值，视觉或行为状态做上下文修正。
- 运动中重点看：
  - 心率区间（热身/燃脂/高强度）
  - 恢复速率（停止后 1~3 分钟回落）

## 5. MVP（8~12 周）实施路线图

### 阶段 1（第 1~2 周）
- 桌宠 UI 壳子（常驻、悬浮、提醒气泡）。
- 摄像头和雷达数据接入打通。
- 本地数据库建表。

### 阶段 2（第 3~5 周）
- 学习/工作状态检测（规则版）。
- 运动动作识别（先支持 2 个动作：深蹲 + 俯卧撑）。
- 语音播报通路打通（TTS）。

### 阶段 3（第 6~8 周）
- 语音对话（ASR + LLM）。
- 健康周报（运动量、专注时长、心率趋势）。
- 设置中心（隐私开关、提醒频率、语音风格）。

### 阶段 4（第 9~12 周，可选）
- 个性化模型（根据你的历史数据调阈值）。
- 动作库扩展（开合跳、平板支撑、弓步蹲）。
- 成就系统和连续打卡。

---

## 6. 你最需要优先解决的难点

1. **隐私合规**：摄像头+雷达+麦克风都很敏感，必须默认本地处理、明确告知。
2. **多传感器同步**：时间戳统一非常关键，否则建议会错位。
3. **实时性能**：动作识别要低延迟（<200ms 体感更好）。
4. **误报控制**：先少提醒、再智能提醒，避免“太吵导致弃用”。

## 7. 数据库表结构（MVP 示例）

- `session`
  - `id`, `type`(workout/study/work), `start_time`, `end_time`
- `vitals`
  - `id`, `timestamp`, `heart_rate`, `resp_rate`, `confidence`
- `activity_metric`
  - `id`, `session_id`, `action_type`, `count`, `score`, `calories_est`
- `focus_metric`
  - `id`, `timestamp`, `focus_state`, `duration_sec`
- `coach_event`
  - `id`, `timestamp`, `level`, `message`, `channel`(voice/ui)

## 8. 第一版交互建议（避免做太重）

- 桌宠默认静默，只在必要时发声。
- 每 45~60 分钟提醒一次活动。
- 运动时切换“教练模式”：
  - 实时计数
  - 关键错误才语音提示
  - 结束后生成 30 秒总结

---

## 9. 你可以立即开始的最小闭环（本周可做）

1. 做一个桌面悬浮窗口（桌宠形象 + 文本提示）。
2. 接入摄像头并跑通 MediaPipe Pose。
3. 先只做“深蹲计数 + 膝盖内扣提醒”。
4. 接入 TTS，播报提醒。
5. 结果写入 SQLite，生成当天运动记录。

做到这 5 步，你就已经有可演示版本了。

---

## 10. 预算与硬件建议（简版）

- 摄像头：1080p 30fps 即可。
- 雷达：选有成熟 SDK 和社区资料的模块（比参数更重要）。
- 设备：若要本地实时跑模型，建议至少 16GB 内存；有独显更佳。

---

## 11. 结论

这个项目完全可做，建议采用 **“桌宠交互 + 本地多传感器 + 规则先行 + 模型迭代”** 的策略。
先把 **动作识别 + 语音纠正 + 本地记录** 做成闭环，再逐步上“专注检测”和“个性化健康分析”，成功率最高。

## 12. 主机 / 主控芯片与硬件选型（重点回答）

你这个项目建议采用 **“PC 主机做 AI 推理 + MCU 做传感器桥接”** 的双层方案：

- **上位主机（必选）**：负责桌宠 UI、视觉推理、语音对话、数据库。
- **主控芯片 MCU（可选但强烈建议）**：负责雷达和其他低速传感器采集、时间戳、断线重连，再把标准数据送给主机。

### 12.1 主机怎么选（优先级最高）

#### 方案 A（推荐给大多数人）：Mini PC / 台式机 x86
- CPU：Intel i5/Ryzen 5 及以上（6 核以上更稳）。
- 内存：16GB 起步，建议 32GB。
- GPU：
  - 起步可用核显（MVP 能跑）。
  - 想要更流畅动作识别+语音并发，建议 NVIDIA RTX 3060/4060 及以上。
- 优点：生态成熟，驱动和 Python/Rust/Node 工具链最稳定。

#### 方案 B（低功耗本地部署）：NVIDIA Jetson Orin Nano / NX
- 优点：边缘 AI 一体化，适合后续做成独立设备。
- 缺点：开发门槛和生态兼容成本高于 x86。
- 适用：你明确要做“硬件一体机”原型，而不是先做 PC 软件产品。

#### 方案 C（苹果设备）：Mac mini M2/M3
- 优点：功耗低、日常体验好。
- 缺点：部分视觉/语音库和外设 SDK 适配需要额外验证。
- 适用：你已经在 Mac 生态开发，并能接受适配工作。

### 12.2 主控芯片（MCU）怎么选

如果雷达模块自带稳定 USB SDK，可短期不加 MCU；
但从工程可维护性看，建议加一个传感器网关 MCU：

#### 推荐 1：**ESP32-S3**（首选）
- 优点：成本低、资料多、USB/UART/I2C/SPI 丰富、可做 BLE/Wi-Fi。
- 适合：1~3 个传感器并发采集、边采边传、轻量预处理。

#### 推荐 2：**STM32F4/F7/H7**（工业稳健）
- 优点：实时性强、外设和中断控制更细、长期稳定性好。
- 适合：你后期要做更严谨的产品化硬件。

#### 推荐 3：**RP2040**（原型友好）
- 优点：便宜、上手快。
- 缺点：生态和工业配套不如 STM32/ESP32 全面。

### 12.3 其他硬件建议（可直接下单的方向）

1. **摄像头**
   - 1080p/30fps，UVC 标准 USB 摄像头优先（免驱）。
   - 关注低光表现和自动曝光稳定性。

2. **毫米波/生命体征雷达**
   - 选“有 PC SDK + 串口协议文档 + 示例代码”的型号。
   - 重点不是极限参数，而是稳定输出心率/呼吸和置信度字段。

3. **麦克风与扬声器**
   - 麦克风：双麦降噪 USB 麦或阵列麦（提高唤醒/识别率）。
   - 扬声器：2~5W 桌面音箱即可，注意低噪底。

4. **电源与连接**
   - 传感器建议统一使用带供电余量的 USB Hub。
   - 给雷达和 MCU 预留独立供电冗余，减少随机掉线。

### 12.4 三档硬件组合（按你的阶段选）

- **入门验证档（最快出效果）**
  - 主机：现有笔记本/台式机（16GB 内存）
  - 传感器：USB 摄像头 + 一款带 SDK 雷达
  - MCU：先不加

- **稳妥 MVP 档（推荐）**
  - 主机：Mini PC（i5/R5 + 16/32GB）
  - 传感器：USB 摄像头 + 雷达
  - MCU：ESP32-S3 做网关（统一时间戳/重连）

- **产品化预研档**
  - 主机：Jetson Orin Nano/NX 或 x86 + 独显
  - MCU：STM32（更强实时性）
  - 结构：定制外壳、固定视角、EMI 与散热设计

### 12.5 给你的明确结论（先买什么）

如果你现在就要开始做，建议优先这套：

1. **主机：x86 Mini PC（i5/Ryzen 5，16GB 内存起）**。
2. **主控：ESP32-S3 开发板**（作为雷达/传感器网关）。
3. **外设：1080p UVC 摄像头 + 有 PC SDK 的毫米波雷达 + USB 麦克风/音箱**。

这套方案在开发效率、稳定性和后续可扩展性之间最平衡，能最快做出可演示且可持续迭代的版本。

## 13. 按你指定器件的整机方案（ESP32-S3 + HLK-LD6002 + 2寸圆形触摸屏）

下面给你一版可以直接开工的硬件设计：

### 13.1 角色分工（强烈建议）

- **ESP32-S3 开发板**：传感器网关 + 小屏 UI + 本地轻交互（触摸按钮/模式切换）。
- **HLK-LD6002 雷达**：输出心率、呼吸、在位信息给 ESP32-S3。
- **上位机（Mini PC/笔记本）**：摄像头动作识别、语音对话、数据库与可视化。

也就是：
- 实时底层采集和设备状态在 ESP32-S3 处理；
- 重 AI 运算在上位机处理；
- 两端用 USB CDC（串口）或 Wi-Fi WebSocket 通信。

### 13.2 2寸左右圆形触摸屏建议

你要“2 寸左右、圆形、带触摸”，建议优先找以下规格的现成模组：

- **显示**：2.1 英寸圆形，分辨率 480x480（IPS/AMOLED 都可）。
- **触摸**：电容触摸（I2C 接口，常见触控 IC：GT911/CST816/CST820）。
- **显示接口**：优先 SPI/QSPI（与 ESP32-S3 对接更简单）。
- **供电**：3.3V 逻辑电平优先（避免电平转换）。

> 选型关键词：`2.1 inch round touch display 480x480 SPI ESP32`。

如果你买不到 2.1 寸触摸圆屏，可退一步：
- 1.85 寸圆屏 + 单独电容触摸环；或
- 方形触摸屏做圆形 UI 蒙版（开发最快）。

### 13.3 推荐硬件清单（你这个组合）

1. **主控**：ESP32-S3-DevKitC-1（N8R8 或更高内存版本更稳）。
2. **雷达**：HLK-LD6002（UART 输出）。
3. **圆形触摸屏**：2.1" 480x480 SPI/QSPI + I2C Touch 模组。
4. **音频（可选接在 ESP32-S3）**：
   - I2S 麦克风：INMP441（单麦）或双麦阵列模组
   - I2S 功放：MAX98357A + 4Ω/3W 喇叭
5. **环境传感器（可选）**：SHT30/SHT31（温湿度，用于健康上下文）。
6. **电源**：5V/3A 适配器 + 带开关电源板（给雷达/屏幕留冗余）。

### 13.4 接线拓扑（建议）

- **LD6002 → ESP32-S3**
  - `LD6002 TX -> ESP32 RX`（UART）
  - `LD6002 RX -> ESP32 TX`（UART）
  - `GND 共地`
  - `VCC 按模块规格供电`

- **圆形屏幕（SPI）→ ESP32-S3**
  - `SCLK/MOSI/CS/DC/RST/BL` 接 SPI + 控制 GPIO
  - 刷新率建议先锁 20~30 FPS，避免占满总线

- **触摸（I2C）→ ESP32-S3**
  - `SDA/SCL/INT/RST` 接 I2C + 中断 GPIO
  - 触摸中断触发后再读取坐标，降低 CPU 占用

- **ESP32-S3 ↔ 上位机**
  - MVP：USB CDC 串口（最稳）
  - 二期：Wi-Fi WebSocket（更灵活）

### 13.5 固件与通信协议（直接可用）

ESP32-S3 每 200~500ms 上报一条 JSON：

```json
{
  "ts": 1730000000,
  "heart_rate": 78,
  "resp_rate": 15,
  "presence": 1,
  "radar_conf": 0.92,
  "touch_event": "tap",
  "mode": "study"
}
```

上位机下发控制指令：
- `set_mode(study/workout/rest)`
- `speak(text)`
- `screen(page, payload)`
- `set_brightness(value)`

### 13.6 2寸圆屏上显示什么（建议 UI）

- 主界面：
  - 中心：桌宠表情/动画
  - 外环：心率 + 呼吸实时值
  - 底部：当前模式（学习/运动/休息）

- 触摸手势：
  - 单击：切换信息卡片
  - 长按 1.5 秒：开始/结束一段 session
  - 左右滑动：切换模式

### 13.7 结构与安装建议（很关键）

- 雷达与喇叭要有一定隔离，避免振动干扰。
- 雷达前方尽量使用非金属外壳窗口（ABS/PC），并避免紧贴金属支架。
- 摄像头与雷达尽量同向，但保持 5~10 cm 物理间距，减少互扰。
- 预留散热孔，尤其是屏幕背面和电源区域。

### 13.8 你现在就能执行的采购与开发顺序

1. 先买：ESP32-S3 开发板 + HLK-LD6002 + 2.1 寸圆形触摸屏模组。
2. 第 1 周：打通 LD6002 串口数据解析 + 屏幕显示心率/呼吸。
3. 第 2 周：触摸手势 + 模式切换 + USB 串口上传上位机。
4. 第 3 周：上位机接入摄像头姿态识别，并融合雷达数据提醒。
5. 第 4 周：加语音播报与本地记录，完成可演示闭环。

按这条路线，你会很快得到一个“能看、能测、能提醒、能记录”的桌宠原型。
